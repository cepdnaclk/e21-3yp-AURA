Step-by-Step

How It Works

From the moment a customer sits down to the moment the kitchen gets the order.

01

Customer Detected

PIR motion sensor wakes AURA from idle. The camera activates and OpenCV Haar Cascade begins face detection. Pan & tilt servos align the head with the customer.

02

Browse the Menu

The 7" touchscreen lights up with the full restaurant menu in the detected customer's preferred language. Categories, item photos, prices, and modifiers are all available.

03

Order Placed

Customer confirms the order via touch or voice command. AURA encodes it as JSON with table ID, items, and special requests, then publishes to the MQTT broker over Wi-Fi.

04

Kitchen Notified

The central Flask server receives the order instantly, records it in SQLite, and pushes it to the kitchen display. AURA confirms to the customer and lights change to dining mode.

Inside AURA

Hardware & Software

Hardware Platform
Raspberry Pi 4 Model B (4GB)

Main processing unit — runs all AI, UI, and networking tasks

7" Capacitive Touch Screen

HDMI-connected display for the ordering interface

Raspberry Pi Camera Module V2

8MP Sony IMX219 — face detection and tracking

2× MG996R Servo Motors

High-torque servos for pan (horizontal) and tilt (vertical) head movement

PIR Motion Sensor

Passive infrared sensor for customer presence detection

10,000 mAh Li-ion Battery

Rechargeable pack powering entire unit — ~8h runtime per charge

USB Speaker

Audio output for voice responses and alerts

Software Application
PyQt5 / Kivy Touch Interface

Full-screen GUI for menu browsing, ordering, and mood controls

OpenCV Face Detection

Real-time Haar Cascade detection; PID controller for smooth servo tracking

SpeechRecognition + pyttsx3

Voice activity detection, Google STT, and offline TTS for responses

Paho-MQTT Client

Lightweight pub/sub messaging between robot nodes and central server

Flask REST Server

Centralised order management, kitchen display, and admin dashboard

SQLite Database

Order history, menu management, and session persistence

i18n Multi-Language Layer

Sinhala, Tamil, and English UI switching — fully localised strings
